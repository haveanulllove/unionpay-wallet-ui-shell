## 拍照或手机相册选图 :id=02030801

通过接口实现小程序唤起本机拍照功能，或者从相册选取图片功能。

```js
upsdk.chooseImage({
    maxWidth: '目标图片宽度, 默认500', 可选
    maxHeight: '目标图片高度, 默认1000', 可选
    sourceType: '1|2|3, 仅允许拍照|仅允许从手机相册中选图|拍照或从手机相册中选图都支持, 默认为3'，可选
    success: function (data) {
        if (data) {
            // data.base64 返回目标图片base64编码.
            // data.type 返回目标图片类型 如 jpg|png|gif
        }
    }
});
```
**错误码**

| 错误码 |  说明 |
| ---- |  :----: | 
|1000	|用户信息为空|
|1001	|未获取到小程序接入id|
|1002	|用户未授权|


## 保存图片 :id=02030802

通过接口实现小程序保存图片功能。

```js
upsdk.saveImage({
    type: '01或者02', //01代表图片url,02代表图片的base64编码
```

## 从本地相册选择图片或使用相机拍照(支持多选) :id=02030803

通过新多媒体插件选择图片（支持拍照与相册，多选），返回虚拟文件路径列表，后续通过文件路径进行读取/上传等操作。

```js
upsdk.chooseImages({
    count: '1',  // 最多可选择的图片张数，默认值 1，string
    sizeType: ['original', 'compressed'],  // 图片类型(iOS当数组中全部为compressed才需要压缩，默认原图)，默认值 ['original', 'compressed']
    sourceType: ['camera', 'album'],  // 选择图片的来源，默认值 ['camera', 'album']
    success: function (data) {
        // data.tempFilePaths 图片的本地临时文件路径列表
    },
    fail: function () {
        // 插件调用失败
    }
});
```

**错误码**

| 错误码 | 错误消息 | 解决方案 |
| ---- | ---- | ---- |
| 1000 | 用户信息为空 | 检查用户是否已登陆 |
| 1001 | 未获取到小程序接入id | 检查调用方是否是小程序 |
| 1002 | 用户未授权 | 再次调用当前插件时，用户对小程序进行授权 |
| 1003 | 用户取消操作 | 用户正常交互流程，不需要特殊处理 |
| 1004 | 无摄像头权限 | 用户没开启系统相机权限 |
| 1005 | 没有开启相册权限 | 用户没开启系统相册权限 |
| 1007 | 图片数量超过最大可选数量 | Android通过系统相册选择的图片数量超过接入方传过来的可选数量 |
| 1008 | 当前插件不支持 | 非小程序调用 |
| 1010 | 其他错误 | 其他异常情况，如安全防护等 |
| 1011 | 文件太大 | 选择小于100M的图片文件 |

>注：支持版本 10.2.8 及以上

## 读取文件内容 :id=02030804

读取虚拟文件路径对应的内容，返回 Base64 数据；支持按位置和长度分段读取，isFinish 表示是否已读完整个文件。

```js
upsdk.readFile({
    filePath: '',  // string 必填，文件路径，支持上述选择图片、视频和录音
    position: '',  // string 非必填，指定开始读取文件的位置，单位：byte；读取范围为 [position, position + length)；有效值范围：[0, maxLength - 1]。若不指定，则从文件开始位置读。注：maxLength 为文件总字节长度
    length: '',  // string 非必填，指定读取文件的长度，单位：byte，有效值范围为 [1, 10485760](最大值10mb)；若不指定，则为10mb。（如果分段读取并拼接建议接入方将base64格式转为byte数组拼接）
    success: function (data) {
        // data.data 读取的文件内容 Base64格式（提示：如果图片类型接入方希望直接展示的话，需要加上头，如"data:image/png;base64,"）
        // data.isFinish "0"表示整个文件未读取完成，"1"表示整个文件已经读取完成
    },
    fail: function () {
        // 插件调用失败
    }
});
```

**错误码**

| 错误码 | 错误消息 | 解决方案 |
| ---- | ---- | ---- |
| 01 | 无效参数 {filePath}/length | 请检查入参 filePath 和 length 的取值 |
| 02 | 未知错误 | 读文件出现异常/客户端内存不足时读大文件会报此错误码，实际开发中请注意做兼容处理 |
| 03 | 文件/目录不存在 | 请检查入参 filePath 对应的文件是否存在 |

>注：支持版本 10.2.8 及以上

## 上传本地文件到开发者服务器 :id=02030805

将本地虚拟文件路径对应的文件上传到开发者服务器，支持自定义 header、formData，建议同时上传不超过5个文件。  

```js
upsdk.uploadFile({
    url: '',  // string 必填，开发者服务器地址
    filePath: '',  // string 必填，要上传的本地文件路径
    name: '',  // string 必填，文件对应的 key，开发者在服务端可以通过这个 key 获取文件的二进制内容
    header: {},  // object 非必填，HTTP 请求 Header。其中 content-type 为 multipart/form-data（与 request body 实际格式一致），请勿更改
    formData: {},  // object 非必填，HTTP 请求中的其他数据，每个 key 为字段名，value 为字符串
    timeout: '',  // string 非必填，超时时间，单位 ms
    hideLoading: '',  // string 非必填，是否隐藏 loading 动画，默认 0
    success: function (data) {
        // data.data 开发者服务器返回的数据
        // data.statusCode 开发者服务器返回的 HTTP 状态码
        // data.header 开发者服务器返回的 Header
    },
    fail: function () {
        // 插件调用失败
    }
});
```

**错误码**

| 错误码 | 错误消息 | 解决方案 |
| ---- | ---- | ---- |
| 01 | 未配置域名白名单，无权访问域名 | 请配置域名白名单 |
| 02 | 环境异常 | 在异常运行场景中调用，例如用户已经登出等小程序不可用情况 |
| 03 | 必要参数非法 | 请检查是否有必传项没传、检查是否有字段传错类型 |
| 03 | 缺少参数 ${name} | 传入必填参数 |
| 03 | 文件不存在 | 请检查本地文件是否存在 |
| 04 | 请求失败 | 请检查网络环境是否正常，服务器是否稳定，外网下是否能正常请求 |
| 04 | 服务器响应失败 | 检查服务器响应参数 |
| 05 | 请使用 HTTPS 请求 | 检查请求地址，确保是 HTTPS 请求 |

>注：支持版本 10.2.8 及以上
